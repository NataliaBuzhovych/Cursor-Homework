Перелік основних команд:

  • git config --global user.name - команда за допомогою, якої задаємо ім'я користувача.
  • git config --global user.email - команда за допомогою, якої задаємо e-mail користувача.
  • git init - команда створює порожній локальний Git-репозиторій.
  • git status - команда для перевірки стану файлів у робочому каталозі та індексі.
  • git add file_name - додати файл або декілька файлів в індекс. Команда git add . дадає всі файли у індекс.
  • git commit - команда відправляє вміст буфера в репозиторій.
  • git log - показує історію комітів.
  • git checkout- команда дозволяє переміщатись між гілками.
  • git branch - команда покаже список гілок, також можна за допомогою властивостей команди створити видаляти чи перейменовувати гілки.
  • git push - відправляє внесені зміни на віддалений репозиторій.
  • git pull - команда завантажує та видаляє вміст віддаленого репозиторію та оновлює локальний репозиторій цими змінами.
  • git merge name-branch - зливання бранча name_branch із тикучим бранчом (в якому ми зараз знаходимя).
  • git clone - клонування віддаленого репозиторія.
Файл .gitignore визначає, які файли та папки слід ігнорувати у даному вихідному коді.
  • git revert - найбезпечніший спосіб скасувати операцію без втрати історії коммітів. Команда скасовує дії минулих коммітів , створюючи новий, що містить всі скасовані зміни. Ця команда корисна, коли ви вже запушили зміни у віддалений репозиторій, оскільки вона зберігає незайманим вихідний коміт.
  • git reset - команда скасовує операцію.
  • git rebase - команда переміщує історію всієї гілки над іншою, переписуючи історію проекту новими комітами.
  • git cherry-pick - команда бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з іншої гілки, а не зливати гілку цілком з усіма змінами, що внесені до неї.
  • git stash - команада зберігіє зміни і коли потрібно до них можна повернутись.
